Interview questions

Each question is scored out of 10. The goal is not to hear a textbook answer, but to understand how the person thinks and what they actually notice.

1. Difference Between QA and Testing (0–10)

Question: What is the difference between Quality Assurance (QA) and Testing?

Expected answer: Testing focuses on finding defects and identifying when the system does not work or may fail. QA focuses on prevention – activities that reduce the likelihood of defects appearing in the product.

Follow-up: Ask for examples of QA activities.

  • 0–3: Confuses QA with testing
  • 4–7: Correct theory, limited practical examples
  • 8–10: Clear preventive QA mindset with real examples

2. Observational Thinking / Critical Thinking (0–10)

Question: Show the candidate the car image and ask: Which car is applying the brakes?

  • Answers “Car #2” → 0 points
  • Says it could be any of them or none of them → 10 points
  • Starts asking clarifying questions → 6 points
  • Notices extra details and asks clarifying questions → 8 points

Goal: Evaluate assumptions, questioning skills and critical thinking.

3. White-box vs Black-box Testing (0–10)

Question: Explain White-box and Black-box testing techniques.

Basic explanation: White-box means knowing how the code is implemented and executed. Black-box means testing inputs and outputs without relying on internal implementation.

Advanced answer: White-box testing matters because incorrect logic may still produce a correct-looking output. Example: 2+2 and 2×2 both return 4, but only one is correct in context.

4. Test Design Techniques (0–10)

Question: What test design techniques do you know?

  • Equivalence Partitioning → 3 points
  • Boundary Value Analysis → 3 points
  • Decision Tables → 2 points
  • State Transition Testing → 2 points
  • Exploratory Testing only → partial score
  • Mentioning test levels/types only → 0 points

5. Test Automation – Good Candidates (0–10)

Question: What makes a test a good candidate for automation?

  • “Everything should be automated” → –5 points
  • Regression only → 2 point
  • Repetitive tests executed frequently → 2
  • Bugs worth protecting from regression → 2
  • Critical business functionality → 2
  • Complex or time-consuming flows → 2

6. Tight Deadlines & Reduced Testing Scope (0–10)

Scenario: A new project has very tight deadlines, and the team wants to skip detailed testing. How would you handle it?

  • “I would test only critical functionality” → 0 points
  • Mentions communication → 2–8 points
  • Clearly explains scope, risks, what will not be tested, and shares this with the team → 10 points

7. REST API Testing Knowledge (0–10)

Question: Do you have experience with REST API testing?

Follow-up questions:

  • You send a request and receive a 401 response. What does it mean? → 2
  • What does the POST method do? → 2
  • What is the difference between PUT and PATCH? → 2
  • A GET endpoint returns a list of objects without paging or limits. Is this a problem, and why? → 2
  • A POST endpoint accepts only a valid email. You send an invalid string and receive a 500 error. Is this OK? → 2

8. Session Security & Cookies (0–10)

Task: This is unsecured session app. Credentials (case sensitive): Jon / password

  • Asks to open DevTools and find cookies → 3 points
  • Aks explains why the session cookie (integer value) is weak → 3 points
  • Aks to modifies cookie to impersonate another user → 2 points
  • Aks to explain what HttpOnly means → 2 points (Cookie cannot be accessed via client-side JavaScript)

9. High-Level Test Plan (0–10)

Task: Show the Calculator app Explain that the calculator has both a frontend and a backend. Ask to prepare a high-level test plan. How to test this?

  • Main functionality with different numbers and operations → 2 points
  • Mentions negative scenarios and validation → 2 points
  • FE–BE communication and error handling → 2 points
  • Performance and/or security → 2 points
  • Opens DevTools and suggests API testing → 2 points

10. Security Bug Analysis (0–10)

Show facebook bug: 500 System ErrorScenario: Changing Facebook’s c_user cookie to -1 returns 500, while other values return 400.

  • Why does Facebook crash only when c_user = -1? → 3 points, Likely -1 does not exist in the DB and causes an unhandled server error
  • Why might Facebook not fix this bug? → 2 points, if mentioned at least one: Legacy system, low priority, risky change, protected by other security layers
  • Why is it a problem to return different error codes for existing vs non-existing users? → 5 points

How to use this in real interviews

This is not about being robotic. It is about avoiding random hiring decisions. Good candidates often show their value in how they ask questions, challenge assumptions, and explain risk — not in how confidently they recite terminology.

  • Use follow-up questions when answers sound memorized
  • Reward practical examples, not polished buzzwords
  • Look for preventive thinking, not just bug hunting
  • Pay attention to how the person handles uncertainty

Final notes

These are mandatory questions. Additional role-specific questions are welcome.

Best use: print this document before the interview and write notes directly next to each topic.
